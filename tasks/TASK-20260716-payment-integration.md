**GitHub Issue:** #9 — https://github.com/prog-time/tg-shop/issues/9

## Что делаем
Реализовать модуль `payment`: инициацию оплаты заказа через внешнего провайдера, публичный вебхук `POST /payments/webhook/{provider}` с проверкой подписи провайдера и идемпотентной обработкой по `provider_payment_id`, и чтение статуса платежа. К провайдеру обращается только этот модуль; событие об оплате пишется в `outbox`.

## Пользователь получает
Как **покупатель**, я могу **оплатить заказ и увидеть статус оплаты**, чтобы **завершить покупку**, при этом платёж не теряется и не обрабатывается дважды.

## Готово когда
- [ ] Given заказ, when вызывается инициация оплаты, then создаётся платёж и возвращаются данные для оплаты у провайдера
- [ ] Given вебхук, when приходит callback, then подпись провайдера проверяется; невалидная → отклонение
- [ ] Given повторный callback, when тот же `provider_payment_id`, then обработка идемпотентна (второй раз не меняет состояние/не шлёт повтор)
- [ ] Given успешная оплата, when подтверждена, then событие пишется в `outbox` (уведомление — через worker)
- [ ] Given `GET /payments/{id}`, when свой заказ (initData) или персонал (adminJWT), then возвращается статус

## Как делаем
- Провайдер за интерфейсом (Telegram Payments / YooKassa / Stripe — опционально, проприетарный только внешний).
- Вебхук — отдельная security-схема `providerSignature`; идемпотентность по уникальному `provider_payment_id`.
- Событие об оплате — только через outbox, не напрямую в брокер.
