# TASK-20260716-cart-and-media: Корзина (Redis) и модуль media

**Type:** feature  
**Size:** L  
**Status:** todo
**GitHub Issue:** #7 — https://github.com/prog-time/tg-shop/issues/7

## Что делаем
Реализовать модуль `cart` (корзина покупателя в Redis: получить, добавить/изменить количество, удалить позицию, очистить) и модуль `media` (выдача presigned URL для загрузки в MinIO и регистрация загруженного объекта; байты не проходят через API, загрузка — только для персонала).

## Пользователь получает
Как **покупатель**, я могу **собирать корзину**, а как **персонал** — **загружать изображения товаров/новостей напрямую в хранилище**, чтобы **оформить заказ и вести каталог с картинками**.

## Готово когда
- [ ] Given корзина, when добавляю/меняю/удаляю позицию, then состояние хранится в Redis (не в PostgreSQL) и переживает переоткрытие
- [ ] Given `POST /media/presigned-uploads`, when переданы content-type и размер, then возвращается presigned PUT URL, подписанный от публичного адреса (`S3_PUBLIC_URL`)
- [ ] Given загрузка, when файл превышает лимит типа/размера, then запрос отклоняется
- [ ] Given загруженный объект, when вызывается `POST /media`, then создаётся запись `media`; серверные операции идут по внутреннему адресу MinIO
- [ ] Given не-персонал, when пытается получить presigned URL, then `403`

## Как делаем
- Корзина: ключ по покупателю в Redis, TTL/persist по решению; состав валидируется против каталога.
- Media: S3-клиент только в этом модуле; presign от `S3_PUBLIC_URL`, служебные операции по `S3_ENDPOINT`.

## Файлы
- `backend/internal/cart/**`, `backend/internal/media/**`
- `backend/internal/<gen>` — реализация операций
