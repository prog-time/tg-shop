# TASK-20260716-catalog-read-api: Каталог и контент (чтение витрины)

**Type:** feature  
**Size:** L  
**Status:** todo
**GitHub Issue:** #6 — https://github.com/prog-time/tg-shop/issues/6

## Что делаем
Реализовать read-модули `catalog` и `content` для витрины: категории, товары со списком/фильтрами (фасеты, цена, пагинация, сортировка) и карточкой, доступные фильтры категории, а также список и карточку новостей (`posts`). Значения справочников в ответах резолвятся из id в отображаемый текст; фильтрация — по `product_facets`, не по JSONB.

## Пользователь получает
Как **покупатель**, я могу **просматривать и фильтровать каталог и читать новости**, чтобы **находить товары и оформлять заказ**.

## Готово когда
- [ ] Given `GET /products`, when заданы фасеты/цена/сортировка/пагинация, then выборка идёт по `product_facets` и индексам, не по JSONB
- [ ] Given `GET /products/{id}`, when товар активен, then `attributes` отдаются с резолвом id справочников в текст
- [ ] Given `GET /categories/{id}/facets`, when есть товары, then возвращаются доступные фильтры категории
- [ ] Given каталог, when часто читается, then горячие ответы кэшируются в Redis с инвалидацией при записи
- [ ] Given `GET /posts` и `/posts/{id}`, when пост опубликован, then он отдаётся; черновики не видны на витрине

## Как делаем
- Реализовать сгенерированные операции чтения; DTO из моделей oapi-codegen.
- Кэш каталога в Redis; ключи по фильтрам, сброс при изменениях (запись — в задаче admin).
- Ссылки на изображения — через модуль media (presigned/публичный адрес).

## Файлы
- `backend/internal/catalog/**`, `backend/internal/content/**`
- `backend/internal/<gen>` — реализация read-операций

## Не делаем в этой задаче
