# ADR-005. Набор Go-библиотек и контракт-first через oapi-codegen

**Статус:** Accepted

## Контекст

Стек проекта фиксировал язык (Go, один модуль, три entrypoint-а), но не библиотеки — `docs/architecture.md` и `CLAUDE.md` прямо предписывали обсудить их до кода. Перед стартом реализации (walking skeleton) выбор нужно закрепить: он формирует скелет, который наследует весь бэкенд. Дополнительная развилка: у нас уже есть контракт `docs/api/openapi.yaml` — генерировать сервер из него или писать хендлеры руками.

Ограничение из `CLAUDE.md`: в ядре только свободные лицензии; перед добавлением зависимости проверяется лицензия.

## Решение

Минимальный набор проверенных библиотек, всё — под пермиссивными лицензиями (MIT / BSD / Apache-2.0), совместимыми с требованием свободного ядра:

| Назначение | Библиотека | Лицензия |
|---|---|---|
| HTTP-роутер + middleware | `go-chi/chi/v5` | MIT |
| Драйвер и пул PostgreSQL | `jackc/pgx/v5` (+ `pgxpool`) | MIT |
| Миграции схемы | `pressly/goose/v3` | MIT |
| Redis-клиент | `redis/go-redis/v9` | BSD-2-Clause |
| AMQP / RabbitMQ | `rabbitmq/amqp091-go` | BSD-2-Clause |
| JWT (админка) | `golang-jwt/jwt/v5` | MIT |
| Метрики Prometheus | `prometheus/client_golang` | Apache-2.0 |
| Кодогенерация из OpenAPI | `oapi-codegen/oapi-codegen/v2` | Apache-2.0 |

Логирование — стандартный `log/slog` (JSON в stdout). Конфигурация — stdlib (`os.Getenv` + тонкий загрузчик), без отдельной библиотеки, чтобы не плодить зависимости.

**Контракт-first.** `docs/api/openapi.yaml` — источник правды для HTTP-слоя. Из него `oapi-codegen` генерирует модели и серверные интерфейсы (`StrictServerInterface`); доменные модули реализуют эти интерфейсы. Спецификация правится раньше кода; сгенерированный код в репозиторий коммитится (воспроизводимость сборки без сети).

## Обоснование

- **chi**, а не фреймворк (Echo/Gin/Fiber): совместим со стандартным `net/http`, минимальная магия, тонкий слой middleware — под модульный монолит (ADR-003) это важнее «батареек».
- **pgx**, а не `database/sql`+ORM: прямой доступ к PostgreSQL, лучшая производительность и типы; ORM противоречил бы принципу «владелец схемы и SQL — Go API». Никакого скрытого генератора запросов.
- **goose**: простые SQL-миграции, встраиваются в бинарь через `embed`; api применяет их на старте — он единственный владелец схемы.
- **oapi-codegen** закрывает риск расхождения кода и контракта: типы и сигнатуры хендлеров генерируются, ручная часть — только доменная логика. Kafka-подобные тяжёлые тулчейны не нужны.
- Всё перечисленное — де-факто стандарт экосистемы Go, активно поддерживается, без copyleft-обязательств.

## Следствия

- Версия Go и точные версии библиотек живут в `backend/go.mod`; этот ADR фиксирует *выбор*, а не версии.
- Добавление новой зависимости — с проверкой лицензии и, если она меняет архитектуру (новый слой, брокер, ORM), с новым ADR.
- Смена HTTP-фреймворка или драйвера БД задним числом — новый ADR со статусом `Superseded`.
- Генерация: `go generate ./...` в `backend/` пересобирает код из `docs/api/openapi.yaml`.
